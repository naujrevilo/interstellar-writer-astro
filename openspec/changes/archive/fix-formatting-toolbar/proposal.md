# Proposal: Fix Formatting Toolbar Cursor Position

## Intent

The formatting toolbar actions (bold, italic, etc.) currently ignore the user's cursor position or text selection, often applying the formatting at the end of the document instead. This change will fix the editor's cursor synchronization so formatting is applied exactly at the cursor or wraps the active selection.

## Scope

### In Scope
- Refactor how the `egui::TextEdit` is rendered in `src/ui/editor.rs` to correctly capture `TextEditOutput`.
- Update the `selection` state synchronization using the direct output of the `TextEdit` rather than attempting to reload the state manually via an unreliable widget ID.
- Ensure `apply_format` correctly handles the text insertion and cursor repositioning based on the accurate selection state.

### Out of Scope
- Adding new toolbar formatting buttons.
- Changing the Markdown parser or preview logic.

## Capabilities

### New Capabilities
None

### Modified Capabilities
None (This is a bug fix for existing behavior, no spec-level requirements change).

## Approach

Currently, `src/ui/editor.rs` uses `ui.add_sized` to render the `TextEdit` widget, which consumes the `TextEditOutput` and returns only a generic `Response`. The code then attempts to recover the cursor state manually using `egui::TextEdit::load_state(ui.ctx(), output.id)`. However, `output.id` may not correctly match the internal state ID generated by `TextEdit`, causing the selection to fail to update and defaulting to the end of the document.

**Solution:**
1. In `src/ui/editor.rs`, stop using `ui.add_sized(..., TextEdit)`.
2. Instead, allocate the required size and call `.show(ui)` directly on the `TextEdit` to obtain the `TextEditOutput`.
3. Extract the `cursor_range` directly from the `TextEditOutput` (or its internal `state`) to reliably update `self.selection`.
4. Ensure `pending_selection` is applied correctly before or during the `TextEdit` state update.

## Affected Areas

| Area | Impact | Description |
|------|--------|-------------|
| `src/ui/editor.rs` | Modified | Refactor `TextEdit` rendering to capture `TextEditOutput` and reliably synchronize cursor state. |
| `src/app.rs` | Modified | Minor adjustments to `apply_format` if necessary to handle the accurate cursor positions. |

## Risks

| Risk | Likelihood | Mitigation |
|------|------------|------------|
| Egui layout constraints break without `add_sized` | Low | Use `ui.allocate_ui` or manually set `.desired_width` and `.desired_rows` to maintain the editor's size bounds. |
| Cursor jumps unexpectedly during typing | Low | Ensure `pending_selection` is only applied when an actual programmatic format action occurs. |

## Rollback Plan

Revert the changes to `src/ui/editor.rs` and restore the `ui.add_sized` wrapping if the direct `.show(ui)` approach causes layout regressions.

## Dependencies

- None

## Success Criteria

- [ ] Clicking bold/italic with no text selected inserts the markdown tokens at the cursor.
- [ ] Clicking bold/italic with text selected wraps the exact selection in markdown tokens.
- [ ] The editor layout maintains its correct dimensions.
